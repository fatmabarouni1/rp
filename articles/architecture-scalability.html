☁️ Architecture & Scalability

AWSCloud NativeServerless

L’architecture de Risk Copilot repose sur une logique cloud native : séparer l’interface, l’API, le traitement IA, le stockage documentaire et le monitoring afin d’obtenir une plateforme claire, évolutive et sécurisée.

Vue d’ensemble

Risk Copilot est conçu comme une application web connectée à plusieurs services AWS. L’utilisateur interagit avec une interface frontend. Les requêtes passent par une API, sont traitées par des fonctions backend, puis enrichies par les services d’IA et les bases de données.

Rôle des composants

Interface Web

Permet à l’utilisateur d’envoyer une question, charger un document, consulter une analyse ou parcourir les résultats.

API Gateway

Expose les endpoints de l’application, contrôle l’accès et transmet les requêtes vers le backend.

AWS Lambda

Exécute la logique métier : préparation du prompt, appel au modèle IA, récupération du contexte et formatage de la réponse.

Amazon Bedrock

Fournit les capacités d’IA générative utilisées pour analyser le contenu, synthétiser l’information et générer des recommandations.

Amazon S3

Stocke les documents analysés : politiques internes, rapports d’audit, procédures et fichiers de référence.

DynamoDB

Peut stocker les métadonnées, historiques d’analyse, conversations, risques identifiés et plans d’action.

Scalabilité

L’approche serverless permet d’adapter automatiquement les ressources au volume d’utilisation. Lorsque le nombre de documents ou de requêtes augmente, les fonctions backend peuvent être exécutées à la demande, sans gérer manuellement des serveurs.

Cette architecture est adaptée à un projet portfolio, mais elle peut aussi évoluer vers une version entreprise avec authentification avancée, files de traitement asynchrones, pipeline d’indexation documentaire et observabilité complète.
